# AI 助教 (AI Tutor) UI/UX 規格書

**版本：** 1.0

## 1\. 簡介 (Introduction)

本文件定義了 **AI 助教 (AI Tutor)** 使用者介面的體驗目標、資訊架構、使用者流程及視覺設計規範。這將作為視覺設計與前端開發的基礎，確保提供一致且以使用者為中心的體驗。

本系統透過費曼學習法 (Feynman technique)，利用 AI 語音互動與行為監測技術，將被動學習轉化為其主動的「教學」。

### 1.1 整體 UX 目標與原則

**目標使用者畫像 (Target User Personas)：**

  * **國中生 (核心使用者)：** 需要一個沉浸式、低干擾的環境來練習費曼學習法。痛點包括「心裡懂但講不出來」以及長時間學習的疲勞。
  * **家長 / 教師 (監督者)：** 需要快速、易懂的洞察報告，以了解學生的弱點與專注程度，而不希望面對過於複雜的數據。

**易用性目標 (Usability Goals)：**

  * **低認知負荷：** 學生端介面必須極簡化，將認知資源保留給「思考」和「講解」，而非操作介面。
  * **非阻塞式回饋：** 針對坐姿與專注度的提醒必須溫和（如吐司通知/輕微震動），避免打斷學生的心流狀態。
  * **可執行的洞察：** 儀表板必須提供由 LLM 生成的具體教學建議，而非僅僅呈現原始數據。

**設計原則 (Design Principles)：**

1.  **沉浸式專注：** 移除雜亂元素。「畫布 (Canvas)」與「語音 (Voice)」是主角。
2.  **藍色系信任感：** 使用科技藍 (`#2563EB`) 色調，傳遞專業與冷靜的心理暗示。
3.  **歷程視覺化：** 將抽象的指標（WPM、停頓）轉化為直觀的視覺圖表。

### 2.2 導航結構

  * **學生端 App (平板/桌機)：**
      * **主導航：** 側邊欄（可收折），以最大化畫布空間。圖示包含：🏠 (首頁)、✍️ (做題)、🗣️ (講題)、❌ (錯題)、🏆 (獎勵)。
      * **講題模式：** **無導航欄**。採用全螢幕沉浸模式，僅保留懸浮控制列。
  * **家長/教師端 App (網頁/手機)：**
      * **主導航：** 頂部標籤列 (Top Tab Bar)，用於切換「總覽」、「詳細報告」和「設定」。

## 3\. 使用者流程 (User Flows)

### 3.1 EPHRC 教學循環 (核心流程)

此流程代表學生向 AI 講解數學題目的過程。

```mermaid
graph TD
    Start[選擇題目] --> Init[載入白板]
    Init --> Explain[狀態：講解 Explain (學生口述)]
    Explain -->|沉默/停頓| AI_Process{AI 分析}
    
    AI_Process -->|概念缺失| Probe[狀態：追問 Probe (AI 提問)]
    AI_Process -->|卡住/無語| Hint[狀態：提示 Hint (模糊線索)]
    AI_Process -->|邏輯錯誤| Repair[狀態：修正 Repair (視覺標記)]
    AI_Process -->|完成講解| Consolidate[狀態：鞏固 Consolidate (總結)]
    
    Probe --> Explain
    Hint --> Explain
    Repair --> Explain
    Consolidate --> End[單元總結 & 獎勵結算]
```

### 3.2 具備 OCR 的做題模式

1.  **進入：** 學生選擇「做題模式」。
2.  **輸入：** 點擊「相機」-\> 拍攝紙本題目 -\> **OCR 處理**。
3.  **確認：** 識別出的文字顯示於可編輯區域 -\> 學生確認。
4.  **解題：** 學生在數位白板上書寫。
5.  **提交：** AI 檢查答案。
      * *若錯誤：* 自動加入「錯題診所」並附上原因標籤。

## 4\. 組件庫 / 設計系統 (Component Library)

### 4.1 設計系統方法

我們將使用 **PrimeVue** 或 **Element Plus** 作為基礎組件庫以加速 Vue.js 開發，並自定義 Token 以符合我們的藍色主題。

### 4.2 核心組件

#### **A. 無限畫布 (白板 Whiteboard)**

  * **用途：** 做題與講題模式的主要工作區。
  * **功能：** 鋼筆工具、橡皮擦、無限捲動、網格背景 (`#EFF6FF`)。
  * **狀態：**
      * *一般：* 藍色筆跡。
      * *修正模式 (Repair Mode)：* AI 在錯誤處繪製紅色虛線框。

#### **B. 流式字幕列 (Streaming Subtitle Bar)**

  * **用途：** 語音轉文字的即時回饋。
  * **外觀：** 底部中央的懸浮膠囊樣式。大字號 (24px)。
  * **狀態：**
      * *聆聽中：* 聲波動畫活躍。
      * *處理中：* 三個跳動的圓點。
      * *AI 發言中：* 背景轉為海軍藍 (`#1E3A8A`)，文字高亮。

#### **C. 吐司通知 (Toast Notification - 非阻塞式)**

  * **用途：** 針對坐姿/專注/壓力的溫和提醒。
  * **外觀：** 半透明深色底層，右上角。Emoji + 簡短文字。
  * **變體：**
      * 🐢 坐姿： "坐直一點喔" (3秒後自動消失)。
      * ☕ 壓力： "深呼吸一下" (綠色背景)。

#### **D. 錯題卡片 (Mistake Card)**

  * **用途：** 在錯題診所中回顧錯誤。
  * **元素：** 題目預覽、「原因標籤」(如：\#觀念、\#計算)、「重播」按鈕。
  * **互動：** 點擊「重播」可直接跳轉至教學過程中發生錯誤的確切時間點。

## 6\. 響應式策略 (Responsiveness Strategy)

### 6.1 斷點 (Breakpoints)

| 斷點 | 範圍 | 目標裝置 | 佈局策略 |
| :--- | :--- | :--- | :--- |
| **平板** | 768px - 1024px | iPad / Android 平板 | **主要目標裝置。** 觸控優化控制。 |

| **手機** | \< 768px | 智慧型手機 | **僅限家長/教師監控端。** 堆疊式卡片報告。學生「講題模式」禁用或僅供唯讀。 |

## 7\. 效能與動畫

### 7.1 動效原則

  * **微互動：** 按鈕點擊時有細微縮放 (0.95x -\> 1.0x)。
  * **模糊解鎖 (Blur Reveal)：** 針對提示 (L1/L2/L3)，使用模糊濾鏡，點擊後轉為清晰文字，將「解鎖知識」的過程遊戲化。

### 7.2 效能目標

  * **白板延遲：** \< 16ms (60fps) 用於繪圖路徑渲染。
  * **語音轉文字延遲：** 流式文字應在說話後 200ms 內出現。
